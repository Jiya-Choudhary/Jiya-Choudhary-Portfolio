Git adalah standar industri untuk kontrol versi. Menggunakannya dengan alur kerja yang tepat sangat penting untuk kesuksesan tim.

## Branching Strategies

1. **Git Flow**: Cocok untuk rilis terjadwal.
2. **GitHub Flow**: Sederhana, cocok untuk Continuous Deployment.
3. **Trunk-Based Development**: Cepat, fokus pada integrasi sesering mungkin.
